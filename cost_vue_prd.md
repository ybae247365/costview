# \[Team. MINT\] 프로젝트 'Cost-Vue' 상세 요구사항 명세서 (PRD)

경제활동을 하는 모든 이들이 자신의 지갑을 지킬 수 있도록, 비정형 뉴스를
정형화된 인과관계 데이터로 정제하기 위한 시스템적 요구사항입니다.

------------------------------------------------------------------------

## 1. 데이터 수집 엔진 (A팀: Data Mining)

세상의 정보를 '녹여내기' 위한 원재료 수집 단계의 정밀도 요구사항입니다.

-   **REQ-01 \| 시맨틱 뉴스 추출**\
    키워드 매칭이 아닌 Exa Semantic Search를 통해 경제적 맥락이 담긴
    고품질 뉴스만 선별 수집

-   **REQ-02 \| 다중 소스 교차 검증**\
    동일 사건에 대해 최소 3개 이상의 신뢰도 높은 언론사 데이터를
    수집하여 편향 제거

-   **REQ-03 \| 실시간 지표 동기화**\
    뉴스와 원자재 가격, 환율 API를 연동하여 텍스트 + 수치 데이터 결합

-   **REQ-04 \| 노이즈 필터링**\
    광고, 댓글, 중복 기사 제거 후 Clean Text 추출

------------------------------------------------------------------------

## 2. 추론 및 구조화 엔진 (B팀: Reasoning)

수집된 정보를 2-Step 로직으로 '주조(Minting)'하는 핵심 요구사항입니다.

-   **REQ-05 \| 2-Step 인과관계 추출**\
    \[원인\] → \[중간 파장\] → \[최종 비용\] 구조로 분해

-   **REQ-06 \| 위험도 스코어링**\
    사용자 지출 영향도를 0\~100 점수로 환산

-   **REQ-07 \| 타겟 카테고리 분류**\
    에너지 / 식료품 / 주거·대출 / 모빌리티 / 기호품

-   **REQ-08 \| JSON Schema 정형화**\
    UI에서 바로 사용 가능한 구조로 출력

------------------------------------------------------------------------

## 3. 저널리즘 및 신뢰성 검증 (QA)

-   **REQ-09 \| 근거 제시**\
    모든 결과에 뉴스 원문 및 데이터 출처 포함

-   **REQ-10 \| 역사적 사례 매핑**\
    유사 과거 사건과 비교하여 신뢰도 강화

-   **REQ-11 \| 톤앤매너 제어**\
    감정 배제, Fact-only 스타일 유지

------------------------------------------------------------------------

## 4. 성능 및 운영 요구사항 (Performance)

-   **REQ-12 \| 처리 속도**\
    전체 파이프라인 5초 이내

-   **REQ-13 \| 확장성**\
    하루 1,000건 이상 병렬 처리 가능 구조

-   **REQ-14 \| 업데이트 주기**\
    최소 1시간 단위 데이터 갱신

------------------------------------------------------------------------
