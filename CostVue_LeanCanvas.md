# 🧠 Cost-Vue — 린 캔버스 (Lean Canvas)
> **"뉴스를 읽지 마세요, 당신의 내일 지출을 확인하세요."**  
> 비정형 경제 뉴스 → 정형화된 인과관계 데이터(Trigger-Chain-Cost) → 개인 지출 리스크 관리 AI 서비스  
> 팀장: 김창해 | 과정: 인텔 AI 과정 | 기술 스택: Exa(시맨틱 검색) + Claude 3.5(인과관계 추론 LLM)

## 1. 🎯 문제 (Problem)

> **"사람들은 경제 뉴스를 읽지만, 자신의 지갑에 무슨 일이 일어날지 모른다."**

| # | 문제 정의 | 현재 상태 (Before) | 이상적 상태 (After) |
|---|-----------|-------------------|-------------------|
| P1 | **정보 과부하** | 하루 수백 건의 경제 뉴스 중 개인에게 유의미한 정보를 추출하는 데 평균 40분 이상 소요 | 개인 지출 패턴에 맞는 리스크 정보만 자동 선별 · 요약 |
| P2 | **인과관계 파악 불가** | "국제 유가 상승" → "내 교통비·난방비 상승" 간의 연결고리를 대부분의 사용자가 독자적으로 도출하지 못함 | AI가 Trigger(뉴스 이벤트) → Chain(파급 경로) → Cost(개인 지출 영향)를 자동 정형화 |
| P3 | **사후 대응 지출 관리** | 물가 상승 이후 가계부를 보며 "왜 이렇게 많이 썼지?" 라는 사후 인식에 그침 | 리스크 발생 전 알림 카드로 사전 대응 행동 유도 |

**문제의 시장 규모 근거:**
- 국내 1인 가구 비율 약 34% (2024년 기준), 연간 지속 증가 추세
- MZ세대 경제 뉴스 구독자의 71%가 "뉴스 내용을 실생활에 적용하기 어렵다"고 응답 (자체 가설)
- 가계 지출 변동성 증가: 소비자물가지수(CPI) 변동 폭 확대 추세

## 3. 💡 고유 가치 제안 (Unique Value Proposition)

### 핵심 메시지
```
"뉴스를 읽지 마세요, 당신의 내일 지출을 확인하세요."
```

### UVP 분해

| 구성 요소 | 내용 |
|-----------|------|
| **What** | 경제 뉴스 이벤트와 개인 지출 간 인과관계를 AI가 실시간 정형화 |
| **Who** | 바쁘고, 경제 지식이 부족하지만, 내 지갑을 지키고 싶은 사람 |
| **Why Now** | LLM 추론 기술의 고도화로 처음으로 실현 가능해진 '개인화된 경제 예보' |
| **Proof Point** | 리스크 스코어(0~100) + 지출 영향 예상 금액 + 대응 액션 카드 제공 |

### 리스크 스코어 예시
```
📌 [지출 리스크 알림]
트리거: 국제 유가 WTI +8% 급등 (2025.07.01)
체인:   국내 정유사 → 주유소 공급가 → 소비자 판매가
예상 영향: 🚗 교통비 +12,000원/월 | 🏠 난방비 +8,500원/월
리스크 스코어: 74 / 100 (HIGH)
추천 액션: 이번 주 내 주유 완료 / 대중교통 전환 검토
```

## 4. 🔧 솔루션 (Solution)

### 핵심 기술 아키텍처: Trigger-Chain-Cost (TCC) 프레임워크

```
[Step 1] 데이터 수집 (Exa 시맨틱 검색)
    ↓ 고품질 경제 뉴스 실시간 크롤링
    ↓ 벡터 유사도 기반 관련 기사 클러스터링
    ↓ 중복/저품질 뉴스 필터링

[Step 2-A] 인과관계 추론 (Claude 3.5 - 1차 추론)
    ↓ 뉴스 이벤트 → Trigger 식별
    ↓ 경제 파급 경로 → Chain 정형화
    ↓ 구조화 JSON 출력

[Step 2-B] 개인화 매핑 (Claude 3.5 - 2차 추론)
    ↓ 사용자 지출 프로파일과 Chain 매핑
    ↓ 영향 금액 예측 및 리스크 스코어 산출
    ↓ 대응 액션 카드 생성

[Step 3] 사용자 딜리버리
    ↓ 모바일 Push 알림 (긴급도 기반 타이밍 최적화)
    ↓ 주간 지출 예보 리포트
    ↓ 인터랙티브 대시보드
```

| 솔루션 기능 | 기술 구현 | 사용자 가치 |
|-------------|-----------|-------------|
| **고품질 뉴스 수집** | Exa 시맨틱 검색 API | 노이즈 없는 핵심 경제 정보만 선별 |
| **TCC 인과관계 추론** | Claude 3.5 2-Step Chain-of-Thought | 뉴스-지출 연결고리 자동 정형화 |
| **개인화 리스크 스코어** | 사용자 지출 프로파일 + LLM 매핑 | 나에게만 해당되는 맞춤 위험도 제공 |
| **지출 예보 알림 카드** | Push 알림 + 인터랙티브 카드 UI | 사전 대응 행동 유도 (예방적 소비 관리) |

## 8. 📈 핵심 지표 (Key Metrics)

| 지표 | 정의 | 목표값 | 측정 방법 |
|------|------|--------|-----------|
| **예측 정확도 (Prediction Accuracy)** | TCC 알림 발송 후 실제 지출 영향 발생 여부 일치율 | ≥ 70% | 사용자 피드백 + 실제 물가 데이터 대조 |
| **리텐션율 (D30 Retention)** | 앱 설치 후 30일 시점 재방문 사용자 비율 | ≥ 40% | 앱 분석 도구 (Amplitude/Mixpanel) |
| **리포트 공유 수 (Viral Coefficient)** | 사용자 1명이 생성하는 유기적 공유 건수 | ≥ 0.3회/월 | 공유 버튼 클릭 + UTM 추적 |
| **프리미엄 전환율 (Freemium → Premium)** | 무료 사용자 → 유료 전환 비율 | ≥ 8% | 결제 시스템 내 코호트 분석 |
| **알림 오픈율 (Push Open Rate)** | 발송된 Push 알림 대비 오픈 수 | ≥ 35% | Firebase / OneSignal 대시보드 |

## 9. 🏆 경쟁 우위 (Unfair Advantage)

| 우위 요소 | 내용 | 모방 난이도 |
|-----------|------|-------------|
| **Trigger-Chain-Cost 독자 알고리즘** | 단순 요약이 아닌 '경제 이벤트 → 개인 지출'의 인과 체인을 정형화하는 독자적 추론 프레임워크. 데이터 누적 시 자기강화(Self-Reinforcing) 구조 | 높음 — 알고리즘 설계 + 도메인 지식 복합 필요 |
| **인텔 AI 과정 최적화 기술** | 엣지 디바이스 최적화 추론 기술 적용으로 API 비용 절감 및 응답 속도 우위 확보 | 중간 — 특정 하드웨어 최적화 노하우 |
| **누적 지출 예측 데이터셋** | 서비스 운영 기간이 길어질수록 사용자 피드백 기반 예측 정확도 DB 축적 → 신규 진입자 대비 우위 | 높음 — 시간 의존적 데이터 장벽 |
| **인과관계 정형화 데이터 희소성** | TCC 포맷의 경제 인과관계 데이터는 시장에 존재하지 않음 → B2B API 판매 시 독점적 공급자 지위 | 매우 높음 |

## ⚠️ 치명적 위험 가설 (Riskiest Assumptions) — 린 스타트업 관점

> 아래 3가지 가설이 틀렸다면 비즈니스 모델 전체가 흔들린다.  
> MVP 단계에서 **가장 먼저, 가장 저렴하게** 검증해야 한다.

### 🔴 RA-1. "사용자는 경제 뉴스와 내 지출을 연결해주는 서비스에 기꺼이 돈을 낼 것이다."

| 항목 | 내용 |
|------|------|
| **왜 위험한가** | 사용자가 '알림의 유용성'을 느끼더라도 "무료로 충분하다"고 판단할 경우, 수익화 불가 |
| **검증 방법** | 랜딩 페이지에 프리미엄 구독 결제 버튼 삽입 후 CTR 측정 (Fake Door Test), 목표: 방문자의 5% 이상이 결제 시도 |
| **검증 비용** | 약 50만원 (랜딩 페이지 제작 + 소규모 광고) |
| **검증 기간** | 2주 |

---

### 🔴 RA-2. "Claude 기반 TCC 추론의 정확도가 사용자가 신뢰하기에 충분한 수준(≥70%)이다."

| 항목 | 내용 |
|------|------|
| **왜 위험한가** | 잘못된 예측(예: "유가 상승 → 식비 10% 증가" 예측 후 실제 변동 없음)이 반복되면 신뢰 붕괴 및 이탈 급증 |
| **검증 방법** | 100개 경제 뉴스 이벤트에 대한 TCC 추론 결과를 실제 통계 데이터(통계청 CPI 등)와 수동 대조, 정확도 측정 |
| **검증 비용** | Claude API 비용 약 10만원 + 인력 2명 × 1주 |

### 🔴 RA-3. "사용자는 지출 리스크 알림을 받은 후, 실제로 행동(사전 구매·소비 변경)을 바꾼다."

| **검증 비용** | 베타 참여 소정의 리워드 (커피쿠폰 등) 약 30만원 |
